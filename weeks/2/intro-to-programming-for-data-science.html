Analytic Problem

Between the years 2016 and 2020, \(KN^3\) company recorded the following revenue: 12000, 23000, 15000, 10000, 18000 respectively. How do you help \(KN^3\) understand if it’s 2021 revenue will be in line with recent revenues?

Python as calculator

Probably your first use of python programming language is as a calculator.

We do not know the revenue for 2021. If we should know, we can compare it with say that of 2020 and determine whether it falls short by a certain percentage. However, using the value of just 2020 to compare with that of 2021 does not necessarily make use of all the recent revenues.

A work around will be to find a number that is a representative of all recent revenues and talk about the revenue for 2021 in terms of our new number. In other words, we want to find the mean of the historical revenues.

(12000 + 23000 + 15000 + 10000 + 18000) / 5
15600.0

Mean as Linear Combination - Weighted

This way of computing mean gives us the arithmetic mean. We summed up all the historical revenues and since we have 5 of them, we divide by 5. We can re-write it as

\(\frac{1}{5}(12000) + \frac{1}{5}(23000) + \frac{1}{5}(15000) + \frac{1}{5}(10000) + \frac{1}{5}(18000)\)

0.2*12000 + 0.2*23000 + 0.2*15000 + 0.2*10000 + 0.2*18000
15600.0

Thus, our new number is formed by taking one-fifth of all the historical revenues and summing them up. Each of our recent revenues contributed to the new number by scaling itself down by \(\frac{1}{5}\). A little bit formal way to put this is, our new number is a linear combination of historical revenues.

It is linear combination because only two operations are allowed: multiplication by scalar and addition of the terms. The concept of linear combination is pivotal to most models used in data analysis, e.g. regression.

If we had used different weights (scale factors), we will be talking about a weighted mean instead of arithmetic mean. You see that the scale factor used in arithmetic mean is intentionally chosen to be the inverse (reciprocal) of the number of items you are summing. If you care about a concise mathematical formulation of arithmetic mean:

\[ \frac{1}{n}\sum_{i=1}^{n}x_i = \frac{x_1+x_2+\dots+x_{n-1}+x_n}{n} \]

Another Kind of Mean

Well, we choose to linearly combine our historical revenues but there are other ways to think about using all the historical records to come up with a new value. How about finding the products of all the 5 revenues and taking the 5th root?

(12000 * 23000 * 15000 * 10000 * 18000) ** (0.2)
14943.577121499638

This way of computing mean is geometric mean. See the mathematical representation below.

\[ \left(\prod_{i = 1}^{n}x_i\right)^{\frac{1}{n}} = \sqrt[n]{x_1\times\dots\times x_{n-1}\times x_n} \]

Persistence - Refer to value in memory

If we want to use the result of our geometric mean for instance, we have to compute it again. The initial computation is lost it in the sea of the computer’s memory. To let it persist and hence re-use it, we have to assign it to a variable.

mean_rev = (12000 * 23000 * 15000 * 10000 * 18000) ** (0.2)
mean_rev
14943.577121499638

Be mindful of the name of the variable, Python has rules on how to construct a valid variable name. You will get used to the rules very quickly but try and use a descriptive name that to some extent describes the value it contains. For instance, we can infer from mean_rev that it refers to the mean of revenues.

The assignment operator = is used to assign what is on the right hand side (RHS) to the variable on the left hand side (LHS) of the operator. Python will first compute the expression on the RHS and then assign the result to the variable on the LHS of the operator.

Remember the integer division // that returns an integer after dividing two numbers? A more general way to enforce a numeric value like mean_rev (which is a float) to be an integer is by casting it by wrapping it in int constructor function.

int(mean_rev)
14943

Suppose we are told that the revenue for 2021 is 16000, we can compare it with our mean of the historical revenues. To test if they are equal we use double equal sign mean_rev == 16000 . We are asking if what is on the LHS is equal to RHS. The result is a Boolean - a value with is either True or False. Once again, you can see the data type of a value using the type() function. Note, this is not the same as assigning what is on the RHS to the LHS. While = is an assignment operator, == is a comparison operator.

mean_rev == 16000
False

If Statement

The company \(KN^3\) will regard a revenue greater than the historical average as desirable. We can test if the 2021 revenue is greater than the mean of the historical revenues. Here, I will introduce the if construct (that helps us to make decisions) and the print() function (for displaying information).

if condition_is_true:
  code_block_specifying_what_to_do

We start with the if keyword, provide a test condition (which should always result in a Boolean) and end with a colon. The next line is indented and we write what happens when the condition provided is true. Remember, python uses indentation to define a block of code. The code specifying what should be done will only be executed if the condition provided returns true.

if 16000 > mean_rev:
  print('above typical revenue') 
above typical revenue

We want to print a statement that reads “above typical revenue” if our 2021 revenue is greater than the historical mean. Change the greater than comparison operator to less than and see what happens. The print will not execute because the condition 16000 < mean_rev is not true.

To specify what happens when the condition is false, we can include the else statement.

if 16000 < mean_rev:
  print("below typical revenue")
else:
  print("above typical revenue")
above typical revenue

Realize that the else part does not take a condition but a way to execute a code when the condition in the if statement is False. Congratulations, you just added another programming vocabulary or trick. The if-else statement helps in decision making based on a condition being true or false.

You just came into contact with the string data type in Python. We can do so with either a single ' or double " quote.

Ternary Operator

{[result_when_true] if [condition] else [result_when_false]}

'below typical revenue' if 16000 < mean_rev else 'below typical revenue'
'below typical revenue'

Sequencing Structures and Collection

So far, we have referred to individual revenues. We want to think of the revenues as a sequence (an array of items) instead of individual revenues. This means we need to choose a data type that supports the ability to consider all the revenues as a group. Python has a number of them but let’s choose to represent our data as a list. The items are separated by a comma and enclosed in a square bracket. Try this if you want a constructor way of defining a list list((12000, 23000, 15000, 10000, 18000)) .

revenues = [12000, 23000, 15000, 10000, 18000]
revenues
[12000, 23000, 15000, 10000, 18000]

Using a sequencing data structure like a list to store our revenues give us some super powers. We can employ Python functions that work on iterable objects. Two common ones used is the sum (which adds up the numbers in the list) and len (which finds the number of items in the list). With these two functions, we can compute the arithmetic mean as follows:

sum(revenues) / len(revenues)
15600.0

Object Oriented Approach

class Means:
  def __init__(self, x):
    self.x = x
  
  def product(self):
    result = 1
    for val in self.x:
      result *= val
    return result
  
  def arithmetic_mean(self):
    result = sum(self.x) / len(self.x)
    return result
  
  def geometric_mean(self):
    result = Means.product(self) ** (1 / len(self.x))
    return result
  
  def harmonic_mean(self):
    x_ = [1/val for val in self.x]
    result = len(x_) / sum(x_)
    return result
  
x = Means(revenues)
x.arithmetic_mean(), x.geometric_mean(), x.harmonic_mean()
(15600.0, 14943.577121499638, 14325.259515570935)

Functional Approach

def product(x):
  result = 1
  for val in x:
    result *= val
  return result

def arithmetic_mean(x):
  result = sum(x) / len(x)
  return result

def geometric_mean(x):
  result = product(x) ** (1 / len(x))
  return result

def harmonic_mean(x):
  x = [1/val for val in x]
  result = len(x) / sum(x)
  return result

arithmetic_mean(revenues), geometric_mean(revenues), harmonic_mean(revenues)
(15600.0, 14943.577121499638, 14325.259515570935)
